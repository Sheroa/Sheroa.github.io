Permission denied when using github

  • git
  • github
  • permission

问题简述

趁着离职空挡，看了一些书，做了一些笔记。都放在电脑上。朋友建议搭个博客。查了一下资料，选择了 hexo搭建github pages。搭建很顺利，直到最后一步：
执行 “hexo d”命令，一直提示

解决

1.确认问题缘由

看了other-user是借我电脑用的同学的邮箱，是权限出了问题。
上网查资料，首先看到了github官方给出的答案：
[This error means the key you are pushing with is attached to an account which does not have access to the repository.

To fix this, the owner of the repository (user) needs to add your account (other-user) as a collaborator on the repository or to a team that has write access to the repository.](https://help.github.com/articles/error-permission-to-user-repo-denied-to-other-user/)
这里，给出了错误的解释，以及他默认的一种情景的解决方案：本机的用户想要访问的库不属于他，需要请求权限。
和我所遇到的情况是不一样的。我的情况，更合适的描述是：要修改本机的account。

2.确认是否为ssh方式的问题

继续找资料，能查到的解决方案都是SSH秘钥的。也难怪，git现在基本都用SSH。但，我的SSH秘钥是这次新配的，配置命令如下：

4.http相关的用户名、密码保存在哪里

我们知道，http方式如果没有别的辅助工具，需要每次上传都填写用户名、密码。那么，是什么辅助工具记录了用户名、密码？
查资料，查到了mac下，自动保存git用户名和密码

看到了一个关键词：osxkeychain
osx表示mac的系统啦，那么keychain呢？
是mac自带的秘钥管理系统。
打开keychain，搜索了error提示中的other-user，搜到了两条git开头记录，果断删除。
之后，再运行 hexo d，提示输入用户名、密码，输入了自己的，瞬间搞定。

####小结

  • mac上的key是由keychain管理的。如果想要查看、删除等，可以用它来完成。
  • 自己的电脑，借给别人用之后，特别是提交代码之类，最好查看一下keychain，不是自己的内容，及时删除。以免后患。
